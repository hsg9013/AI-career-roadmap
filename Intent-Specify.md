# Intent - Specify

[정보화 비전·목표]
이 서비스는 대학생이 자신의 진로를 데이터에 근거해 설계하고, 취업 준비 과정을 자동으로 관리받을 수 있도록 돕는 플랫폼입니다.

기존의 채용 정보 서비스가 공고를 단순히 나열하는 데 그쳤다면, 우리 서비스는 실제로 취업에 성공한 선배들의 (개인 정보를 지운) 합격 경로 데이터와 현직자의 직접 피드백을 인공지능으로 결합합니다. 이를 통해 학생 한 명 한 명에게 맞는 진로 방향과 준비 계획을 제시합니다.

핵심 목표는 네 가지입니다. 첫째, 학생의 현재 역량과 목표 직무가 요구하는 역량의 차이를 진단합니다. 둘째, 선배들의 경로를 바탕으로 시기별 합격 로드맵을 설계합니다. 셋째, 실무 과제(미션)와 인공지능·현직자의 결합 피드백으로 실전 역량을 키웁니다. 넷째, 학생의 활동 기록을 자동으로 정리해 이력서와 포트폴리오로 만들어 주고 진척 상황을 점검합니다.

수익은 네 가지 경로로 만듭니다. 학생 대상 유료 멤버십과 단건 결제, 대학·기업 대상 라이선스와 채용 수수료, 채용 공고 광고, 그리고 외부 교육·자격증 콘텐츠 제휴 수수료입니다.

[환경·개선 동인]
이 서비스가 필요한 이유는 학생과 시장이 겪는 여러 어려움 때문입니다.

학생들은 "무엇을, 언제, 어떤 순서로 준비해야 합격하는지"에 대한 구체적인 정보를 얻기 어렵습니다. 포트폴리오와 자기소개서는 매번 처음부터 다시 정리해야 하는 부담이 크고, 실무자 관점의 제대로 된 피드백을 받을 곳도 마땅치 않습니다.

한편 대학교 취업지원센터는 학생들의 진로 준비 현황을 체계적으로 관리하고 취업률을 높여야 하는 과제를 안고 있습니다. 기업은 검증된 인재를 일찍 발굴하고 싶어 합니다.

이 서비스의 이용자는 크게 네 부류입니다. 진로를 준비하는 학생(주 사용자), 학생을 관리하려는 대학(B2G), 인재를 찾는 기업과 멘토링하려는 현직자(B2B), 그리고 자신의 합격 경로 데이터를 제공하는 선배입니다.

규제 측면에서는 한국의 개인정보보호법과 정보통신망법을 반드시 지켜야 합니다. 학적·성적 같은 민감한 정보를 다루기 때문입니다. 기술 측면에서는 웹과 모바일 앱(iOS·안드로이드)을 함께 출시하며, 결제·메일·푸시 알림은 외부 전문 서비스를 연동해 처리합니다.

[갭 분석·개선 기회]
001·002 두 번의 구축으로 12개 기능 묶음(계정·진단·로드맵·문서·미션·알림·동의·대학·기업·결제·외부데이터·운영)의 구조는 모두 갖춰져 p16.sumzip.com에 라이브로 동작합니다. 이제 남은 갭은 "기능이 없다"가 아니라 핵심 가치의 알맹이가 아직 시범(stub) 수준이라는 데 있습니다. 003은 이 알맹이를 실제 동작으로 전환하는 단계입니다.

첫째, 추천·문서·미션 피드백이 규칙 기반 임시 동작(AI 미연동)에 머물러 있습니다 → 실제 AI 모델을 연동해 "데이터 기반 맞춤"의 본래 가치를 채웁니다. 둘째, 회원가입의 관심 산업·희망 직무 선택지와 진단 기준이 5개 직무(IT 4 + 금융 1)에 그쳐 빈약합니다 → 주요 산업 10여 개·대표 직무 수십 개 수준으로 직무·산업 사전을 확충하고 직무별 핵심 역량 키워드를 채워, 진단·추천이 더 많은 학생에게 들어맞게 합니다. 셋째, 결제·정산이 시범 처리(실결제 미연동) 상태입니다 → 멤버십·단건·멘토/선배 정산의 실제 돈 흐름을 가동합니다. 넷째, 알림이 앱 내 알림만 작동합니다 → 모바일 푸시·이메일을 더해 3채널을 완성합니다. 다섯째, 채용·자격증·공모전 정보가 시범 데이터 수준입니다 → 외부에서 일 단위로 모으는 수집 경로를 만듭니다. 여섯째, 소셜 로그인·학교 이메일 검증이 미완입니다 → 네이버 로그인과 .ac.kr 검증을 더합니다. 일곱째, 모바일 앱이 미배포(웹만 라이브) 상태입니다 → iOS·안드로이드 스토어 출시로 "웹+모바일 동시 제공"을 채웁니다. 여덟째, 운영 안정성이 미검증입니다 → 월 99.5% 가용성·일일 백업·24시간 복구를 자동화·이중화로 확보합니다.

[TO-BE 목표 기능]
앞으로 만들 기능은 크게 열두 개 묶음으로, 모두 52개의 세부 요구사항으로 구성됩니다.

계정과 권한 관리(학생·현직자·기업·대학·운영자 5종 역할), 프로필 입력과 역량 갭 진단, 선배 경로 기반 합격 로드맵, 활동 자동 정리와 이력서·자기소개서·포트폴리오 자동 생성, 실무 미션과 결합 피드백, 진척 점검과 알림, 개인정보 보호와 동의 관리, 대학용 기능, 기업용 기능, 결제와 보상, 외부 채용·자격증 데이터 수집, 운영자 관리 기능입니다.

이 기능들은 한꺼번에 만들지 않고 우선순위에 따라 단계적으로 출시합니다. 가장 먼저(P1) 역량 갭 진단을 내놓고, 다음으로(P2) 로드맵 추천과 문서 자동화를 더합니다. 그 뒤(P3) 미션·알림·멘토 정산·대학 기능을 붙이고, 마지막으로(P4) 기업 매칭·결제·선배 데이터 기부 기능을 완성합니다.

[TO-BE 데이터 아키텍처]
앞으로 갖출(그리고 이미 상당 부분 구축한) 데이터 구조와 보호 방식입니다.

데이터는 성격에 따라 세 곳에 나누어 저장합니다. 회원·활동·로드맵처럼 서로 연결된 핵심 데이터는 MariaDB에, 이력서·미션 제출물 같은 파일은 MinIO에, 빠른 조회용 임시 데이터와 작업 대기열은 Redis에 둡니다. 전체 데이터는 35개 항목으로 설계되어 학생·추천엔진·미션피드백·알림·파트너·결제·동의 영역을 아우릅니다.

보호 장치는 특히 신경 써서 설계했습니다. 선배 데이터는 개인을 식별할 수 없도록 익명화하며, 같은 조건의 사람이 최소 5명 이상 모여야만 추천에 사용합니다(k-익명성). 계좌번호·주민번호 같은 민감 항목은 암호화해 저장하고, 동의 내역은 바꿀 때마다 새 기록을 남겨 추적 가능하게 합니다. 대학 공유 동의는 '공유 안 함 / 통계만 / 개인 단위' 3단계로 세분화하며, 졸업 후 일정 기간이 지나면 자동으로 익명화·삭제합니다.

[TO-BE 애플리케이션·기술 아키텍처]
앞으로 갖출 애플리케이션과 기술 구조입니다.

전체는 하나의 저장소 안에 다섯 개의 작업 공간으로 나눕니다. 서버(백엔드), 웹 화면, 모바일 앱, 웹과 앱이 함께 쓰는 공통 부품, 그리고 화면과 서버가 주고받는 데이터 형식 정의입니다. 웹은 Vue로, 모바일 앱은 같은 화면 코드를 Capacitor로 감싸 iOS·안드로이드에 함께 올립니다. 덕분에 같은 기능을 두 번 만들 필요가 없습니다.

주요 기술 선택은 여덟 가지로 정리됩니다. 추천 엔진은 인공지능과 규칙 기반을 함께 쓰고, 파일은 MinIO에 저장하며, 결제와 정산은 PortOne을, 메일과 푸시 알림은 네이버 클라우드와 FCM·APNs를 씁니다. 로그인은 보안 토큰 방식을 채택하고, 대학 기능은 통계용과 개인용 권한을 분리하며, 멘토 보상은 정액과 수수료 두 방식을 모두 지원합니다.

성능 목표도 정해 두었습니다. 로드맵 생성은 2초 안에, 일반 조회는 0.2초 안에 응답하고, 모바일 앱은 3초 안에 켜지며, 푸시 알림은 5초 안에 도착하도록 합니다.

[기대효과·성공지표]
서비스가 잘 작동하는지 판단할 구체적인 목표 수치(성공 지표)를 정해 두었습니다.

학생 경험 측면에서는, 처음 정보를 입력하고 첫 진단 결과를 받기까지 평균 15분 이내가 목표입니다. 가입 후 7일 안에 핵심 행동 세 가지를 모두 해 보는 학생이 절반 이상, 추천 로드맵을 한 가지라도 완료하는 학생이 70% 이상이 되도록 합니다. 추천을 거부하는 비율은 30% 이하로 낮추고, 자동 생성한 포트폴리오를 실제로 활용하는 학생은 90% 이상을 목표로 합니다. 미션 피드백은 5영업일 안에 전달되는 비율이 80% 이상이어야 합니다.

데이터와 보안 측면에서는, 선배 데이터의 익명화 기준을 100% 지키고 정보 유출 사고는 0건을 유지합니다.

수익 측면에서는, 무료 사용자가 90일 안에 결제로 이어지는 비율 8% 이상, 멤버십 해지율은 월 5% 이하, 학기말 알림을 받은 학생의 재방문율 60% 이상을 목표로 합니다.

[이행 과제·우선순위]
001·002로 준비·기반 단계와 P1~P4 전 기능의 구조 구현이 끝났고, 웹 서비스가 운영 도메인에 떠 있습니다. 003의 과제는 새 기능을 더하는 것이 아니라 이미 만든 기능의 실연동과 안정화입니다. 우선순위는 다음과 같습니다.

1순위(핵심 가치 직결, 외부 키 의존이 적어 먼저 착수): ①추천·문서·미션 피드백의 실제 AI 연동, ②회원가입 관심 산업·희망 직무 선택지와 진단 기준이 되는 직무·산업 사전 확충(직무별 역량 키워드 포함). 2순위(수익 실현): ③결제·멘토/선배 정산의 실결제 전환. 3순위: ④알림 3채널 완성(모바일 푸시·이메일), ⑤외부 채용·자격증·공모전 수집 파이프라인, ⑥소셜 로그인·학교 이메일 검증. 4순위: ⑦모바일 앱 스토어 배포. 전 과정과 병행: ⑧운영 안정성 — 일일 백업·24시간 복구 절차, 단일 장애점 보완(예비 서버), 모니터링·알림 체계.

유의할 의존 관계: 결제 실연동은 멘토 정산 데이터 구조 위에서 진행합니다. AI 연동(①)은 모델 키·호출 비용·실패 시 규칙 기반 대체(폴백) 정책을 먼저 정한 뒤 켜며, 확충된 직무·산업 사전(②)을 진단·추천의 기준으로 함께 활용합니다.

[이행 로드맵]
현재(2026년 6월) 001(역량 갭 진단 MVP)과 002(P2~P4 전체 기능)의 구현이 끝나 웹 서비스가 p16.sumzip.com에 라이브입니다. 구조는 완성됐고, 외부 연동·AI·콘텐츠·운영 안정성이 남은 과제입니다.

003의 흐름은 다음과 같습니다. [1] 추천·문서·미션 피드백에 실제 AI를 연동하고, 같은 초반에 직무·산업 사전을 확충해(회원가입 선택지·진단 기준) 핵심 가치와 적용 범위를 함께 넓힙니다 → [2] 결제·정산을 실결제로 전환합니다 → [3] 알림 3채널(푸시·이메일)·외부 데이터 수집·소셜 로그인을 더합니다 → [4] 모바일 앱을 iOS·안드로이드 스토어에 배포합니다 → [5] 운영 안정성(일일 백업·24시간 복구·이중화)과 성능·접근성을 다듬어 마무리합니다.

핵심 원칙은 002와 같습니다. '한 번에 다 켜지 않고, 검증 가능한 단위로 차례대로'. 특히 외부 연동(AI·결제·푸시)은 키·비용·장애 시 대체 동작을 단계마다 확인하며 점진적으로 활성화합니다. 반면 직무·산업 사전 확충은 외부 키·비용 의존이 없어 [1]에서 즉시 착수합니다.

[위험·제약·거버넌스]
예상되는 위험과 그 대비책, 그리고 운영 규칙을 정리합니다.

주요 위험과 대응은 다음과 같습니다. 초기에 선배 데이터가 없는 문제는 데이터 제공자에게 보상을 주어 모읍니다. 추천이 인기 직무로 쏠리는 편향은 거부 이력을 학습하고 성별·나이·출신 학교 같은 민감 정보를 추천에서 빼는 방식으로 줄입니다. 정보 유출 위험은 권한 관리·접근 기록·암호화로 막습니다. 데이터베이스가 한 대뿐이라 생기는 장애 위험은 향후 예비 서버를 두어 보완합니다. 현직자 피드백이 늦어질 때는 다른 멘토에게 재배정하거나 인공지능 피드백으로 대체합니다. 채용 정보가 오래되면 '최신 아님'을 표시합니다.
