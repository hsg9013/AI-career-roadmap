# Research: 004 — 가치사슬 앞단·생태계·수익 보강

**Date**: 2026-06-10 | **Branch**: `004-intent-specify`

004는 신규 기술 스택 도입이 거의 없고(외부 SDK 추가 없음), 대부분 기존 001~003 모노레포 모듈 확장이다. Phase 0의 NEEDS CLARIFICATION은 `/speckit.clarify`에서 모두 해소되었으므로, 본 문서는 각 갭(G1~G6)에 대한 결정·근거·대안을 정리한다.

## R1. 활동·스펙 기록 (G1)

- **Decision**: 기존 `modules/activities` 를 확장하고 보유 스펙(자격증·수상·인턴·아르바이트)을 별도 엔티티로 추가. 입력은 수기(폼·키워드 선택) 중심, 온보딩에서 안내하되 비차단(스킵 가능)·마이페이지 상시 추가. 문서·진단·로드맵이 이 기록을 입력으로 소비.
- **Rationale**: "먼저 활동을 기록하라"는 안내가 실제 동작하려면 입력 경로가 선행되어야 한다. 비차단으로 두어 가입 이탈을 막고, 빈 입력 시 안내로 데이터 확보를 유도.
- **Alternatives**: ①필수 차단 입력 — 이탈 위험으로 기각. ②외부 LMS 자동 연동 — 범위·비용·동의 복잡도로 004 범위 외.

## R2. 직무별 합격 경로 표본 확충 (G6)

- **Decision**: `alumni` 합격 경로 표본을 직무당 10건 시드. 같은 직무 10건이면 k-익명성(≥5)을 안정 충족. 시연용 임의 계정(학생/선배/멘토)은 별도 시드로 생성하고 목록 공유.
- **Rationale**: 표본 빈약으로 개인화 추천이 일반 가이드로 떨어지는 문제를 해소. 10건은 k≥5 + 다양성 확보의 실무적 최소선.
- **Alternatives**: 외부 실데이터 수집 — 동의·법적 부담으로 초기 부적합. 5건 — k 경계라 코호트 분할 시 미달 위험.

## R3. 직무 맞춤 미션 (G2)

- **Decision**: 직무별 기본 미션 세트(직무당 ≥3) + 매칭 멘토 출제. 멘토 출제가 있으면 우선, 없으면 기본 세트.
- **Rationale**: 현재 동일 미션 문제를 직무 코드 기준 세트로 해결하면서, 현직자 결합 가치(003)를 살린다.
- **Alternatives**: 멘토 출제 전용 — 멘토 풀 부족 시 공백. 전면 AI 생성 — 품질 편차·비용으로 기본 세트를 우선.

## R4. 로드맵 추천 설명가능성 (G3)

- **Decision**: 로드맵/진단 결과에 평이한 추천 근거(예: "같은 직무 합격 선배 N명 경로 + 직무 요구역량 비중 기반")와 '가중치' 설명 문구를 부가. 추가 입력이 추천을 어떻게 개선하는지 안내. 수치(가중치 값)는 기본 숨김, 설명 문장 우선.
- **Rationale**: 결과의 의미·데이터 출처가 안 보여 신뢰가 떨어지는 문제 해결. 기존 `gateway.ts`/`roadmap` 가 이미 source/ai_summary를 반환하므로 rationale 필드만 부가하면 됨.
- **Alternatives**: 원시 가중치 노출 — 일반 사용자 혼란으로 기본 숨김.

## R5. 다각 파트너십 생태계 (G4)

- **Decision**: 대학·기업·현직 멘토·교육/활동 플랫폼·기술 협력사를 역할별로 연결. 파트너 계정은 초기 운영자 수동 등록·역할 계정 발급(셀프가입 X). 대학=학생 데이터·교육과정/상담 연계, 기업=미션·채용 제공, 교육 플랫폼=콘텐츠·활동 정보(feeds 연계). 동의 범위·권한 분리 적용.
- **Rationale**: 데이터 수급·취업 연계·수익의 토대. 초기 신뢰·검수를 위해 수동 등록이 안전.
- **Alternatives**: 셀프가입 — 검증·악용 위험으로 후순위.

## R6. 수익 구조 구체화 (G5)

- **Decision**:
  - 멤버십: 무료(Free)/프리미엄(Premium 구독) 2단계. 무료=진단 기본·로드맵 조회·활동/스펙 기록·앱 내 알림·AI 문서 횟수 제한. 프리미엄=로드맵 심층 분석·포트폴리오 진단·취업 준비 패키지·AI 문서 무제한·푸시/이메일. 멤버십 화면에 항목별 비교표.
  - 실무 과금: AI 자소서 첨삭·이력서 피드백·현직자 멘토링·실전 과제 평가의 단건 수수료.
  - 타겟 채용 광고: 자체 광고 슬롯(내부 관리), 학생 직무·동의 기준 노출, 광고주 과금. 외부 광고 네트워크 연동은 범위 외.
  - 제휴 배너: 외부 교육 제휴사 배너·할인, 클릭/전환 자체 집계 → 제휴 수수료. 외부 정산 연동 범위 외.
  - B2B/B2G: 대학 SaaS 연간 라이선스(동의 범위별 통계/개인), 기업 채용 성사 수수료.
- **Rationale**: 결제·정산 구조(003)와 동의·권한(001~003) 위에서 내부 로직으로 시작해 검증 후 외부 연동을 단계적으로 추가(점진 활성화).
- **Alternatives**: 외부 광고/정산 즉시 연동 — 키·비용·정산 정합 부담으로 자체 집계 우선.

## R7. 용어 정비

- **Decision**: 선배 데이터 '기부' → '합격 경험 공유'(기여자=공유 선배, 대가=공유 보상). 화면·데이터·문서 전역 치환, 잔존 0 검증.
- **Rationale**: '기부'는 자선 뉘앙스로 커리어 도메인과 위화감. '공유'가 자연스럽고 보상 구조와도 양립.
- **Alternatives**: '커리어 기여'·'경로 나눔' — 후보로 검토했으나 '합격 경험 공유'가 가장 직관적.

## 종합

- 신규 외부 의존성 없음(내부 로직·시드·UI 중심). 외부 광고/제휴 정산 연동은 명시적으로 범위 외.
- 모든 NEEDS CLARIFICATION 해소됨 → Phase 1 진행 가능.
