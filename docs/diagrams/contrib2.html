#분석·설계 영역주담당기여내용 (무엇을 / 어떻게)근거 (DDBM 캔버스 블록 · ISP)
1정보화 비전·목표 정의황성권 서비스 방향을 "대학생이 진로를 데이터에 근거해 설계하고 취업 준비를 자동 관리받는 플랫폼"으로 정리. 4대 핵심 목표(① 역량 갭 진단 ② 선배 합격 경로 기반 로드맵 ③ 실무 미션 + AI·현직자 피드백 ④ 활동 기록의 문서 자동화) 정의 「정보화 비전·목표」 블록 · ISP §1
2환경·개선 동인 정리황성권 "무엇을·언제·어떤 순서로 준비할지 알기 어렵다 / 포트폴리오·자소서 반복 작성 / 실무자 피드백 부재"라는 핵심 동인을 정리하고, 대학·기업·현직자·교육활동 플랫폼이 연결되는 파트너십 구조를 설계 「환경·개선 동인」 블록 · ISP §2
3AS-IS 현황·한계 분석황성권 현행 방식(채용 포털·후기 산재, 막연한 자기 역량 판단, 워드·한글 수작업 포트폴리오, 개인 메모·캘린더 진척 관리)의 한계를 정리 → "단순 채용 공고 사이트가 아니라 취업 준비 통합 관리 시스템"이라는 방향 도출 「AS-IS 기능 현황」 + 「AS-IS 문제점·한계」 블록 · ISP §3·§4
4갭 분석·개선 기회 도출황성권 잔여 문제(활동·스펙 입력 단계 부재, 직무 맞춤 미션 부족, 로드맵 추천 근거 설명 부족, 파트너십 생태계 미구현, 수익 구조 미구체화)를 정리 → 실제 구현(활동·스펙 입력, 로드맵 근거 설명, 미션, 멤버십, 광고·제휴)으로 연결 「갭 분석·개선 기회」 블록 · ISP §6
5위험·제약·거버넌스 정리황성권 운영 제약 정의: 개인정보보호법·정보통신망법 준수, 선배 데이터 k-익명성 최소 5명, 광고·제휴 표기, 결제 테스트모드 표시, 토큰 저장 시 민감정보 저장 금지 「위험·제약·거버넌스」 블록 · ISP §13
